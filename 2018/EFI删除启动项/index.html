删除双系统后的收尾问题

发表于 2018-05-21 | 分类于 日常 ， 系统问题

问题由来

由于尝试装了一个Linux双系统，但是装上去之后由于不知道的原因无法连上校园网而且也无法打开终端，只有咬咬牙把系统给删了，问题是装在和Windows10相同硬盘中，共享了一个EFI盘，无法删除干净，在启动系统的时候总会优先进入Linux，更改启动优先项之后，可以直接进入系统，但是进入BIOS时看着很不爽，就查找了解决方法。

解决方法

问题扩展

怎么删除双系统？

在Windows10下直接找到对应的分区，格式化或者删除就好

主要的解决方法:

  1. 查看电脑分区信息，确定EFI分区的分区号

    • Windows + R打开“运行”
    • 输入diskmgmt.msc打开 磁盘管理（也可以在控制面板等位置进入）
    • 在状态 栏中找到 EFI系统分区 并记录分区号（“卷”栏下对应的最后一个数字）

  2. 利用管理员权限打开cmd，输入diskpart

    • 输入list disk展示所有磁盘信息，一般0号磁盘为系统盘

    • 输入select disk 0 选择使用的磁盘（选择EFI系统分区所在的磁盘，此处默认盘号为 ’0‘ )

    • 输入list partition 查看分区列表以确定具体分区

    • 输入select partition 1根据之前查看的分区度确定分区（一般大小为260MB，windows下，我的电脑上是 ‘1’ 号分区）

      在这里有另一个方法省略一下步骤，直接输入delete partition override 强制删除EFI分区，但是可能会造成问题，未尝试这个。

    • assign letter = p 为win10的EFI分区分配盘符，字母A~Z都可以，不区分大小写，但是不能够和已有盘符冲突！！

    • 打开电脑，找到并打开新配的盘 （此处为 ‘p’ 盘），找到unbutu文件夹全删掉！

      在此处可能会出现不能够打开p盘的问题，若百度的结果无法解决，推荐直接在第一步之前先进入系统的安全模式，再进行操作。

      如何进入安全模式看这个win10电脑系统如何进入安全模式

    • 返回终端，remove letter = p 删除配置的盘符。

    • 结束
